## Summary: Key SERP Patterns Across All Queries

### Dominant competitors
| Domain | Appearances in top 3 | Strength |
|--------|----------------------|----------|
| visitbergen.com | 4/10 queries | Domain authority, official tourism |
| ticketmaster.no | 3/10 queries | Event schema, ticket inventory |
| facebook.com | 3/10 queries | Community, freshness |
| tripadvisor.com | 2/10 queries | Reviews, listicle format |
| Travel blogs | 3/10 queries | Backlinks, listicle SEO |

### SERP features observed
- **Event carousels/rich results**: Triggered by Ticketmaster, Songkick (Event schema markup)
- **Facebook groups/pages**: Rank for community/niche queries (quiz, stand-up, "i dag")
- **Numbered listicles**: Dominate tourist-intent queries ("ting a gjore", "free things")
- **No featured snippets observed**: These queries trigger list/calendar results, not paragraph snippets
- **No map packs observed**: Event queries don't trigger local map results in these searches
- **No PAA boxes reported**: Likely present but not captured in search API results

### Biggest opportunities for gaari.no

1. **"quiz bergen"** — Fragmented SERP, no aggregator. A dedicated quiz collection page could rank #1.
2. **"arrangementer bergen i dag"** — Freshness advantage. gaari.no has daily-updated data.
3. **"bergen events today"** — Weak English-language competition. Eventbrite returns wrong city.
4. **"messer bergen 2026"** — Low competition niche. Easy win with a collection page.
5. **"konserter bergen denne uken"** — Temporal query where daily-fresh data beats static pages.

### Content format priorities

1. **Event schema markup** on every event page (mandatory — this triggers Google event carousels)
2. **Auto-updating temporal pages** for "i dag", "denne uken", "denne helgen" queries
3. **Numbered listicle format** for evergreen/tourist queries ("ting a gjore", "free things")
4. **FAQ schema** on collection pages answering natural-language questions
5. **Weekly schedule format** for recurring events (quiz nights, stand-up shows)

### Technical priorities

1. Verify Event structured data is rendering correctly (test with Google Rich Results Test)
2. Ensure ISR caching doesn't prevent Googlebot from seeing fresh daily content
3. Implement hreflang properly between /no/ and /en/ pages
4. Add FAQ schema to collection pages
5. Create or optimize collection pages for: quiz, stand-up, messer, konserter-denne-uken, gratis-i-dag
